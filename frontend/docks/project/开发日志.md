**因浏览器与Next的缓存策略导致的组件不重新挂载问题**

- 浏览器直接从内存中恢复页面快照（bfcache特性），而不重新加载HTML/CSS或执行JavaScript

- 因此事件监听器、定时器、useEffect 等初始化逻辑不会重新执行

- 当前暂时使用 replace 跳转解决

# Day 8

1. **实现输入功能 UI 按钮**

完成了输入区域的 UI 按钮组件开发，支持用户输入交互。按钮样式与整体设计系统保持一致，后续可根据业务需求扩展具体功能逻辑。

2. **实现历史消息区域的 Flex 布局滚动**

在 Flex 布局下实现了历史消息记录的溢出滚动效果。核心要点如下：
- 使用 `flex-1` 让历史消息区域占满剩余空间
- 关键设置：添加 `min-h-0` 覆盖 Flex 子项默认的 `min-height: auto` 属性，避免内容撑开容器导致滚动失效
- 配合 `overflow-y-auto` 实现内容超出时的滚动效果

该方案无需 JavaScript 计算高度，纯 CSS 实现动态高度适配。

3. **实现响应式侧边栏与主内容区布局**

完成了页面级的响应式布局适配：
- 侧边栏响应式：监听窗口宽度变化（断点 1024px），窗口小于断点时自动收起，大于断点时自动展开
- 用户手动操作优先级：用户手动收起侧边栏后，不再受自动展开逻辑影响，直至用户主动展开，保证交互预期
- 主对话区域：根据侧边栏状态自适应宽度，实现流畅的布局切换
