### 数据流分层

| 层级 | 技术 | 职责 |
|------|------|------|
| **UI 状态层** | Zustand (5 stores) | 侧边栏展开、对话/知识库模式切换、深度思考开关、知识库选择 |
| **服务端缓存层** | React Query (TanStack) | 会话列表、知识库列表、文档列表、聊天历史 —— 自动缓存/失效/乐观更新 |
| **流式通信层** | AI SDK (`useChat`) | 实时流式聊天，SSE 协议 |
| **API 层** | Next.js Route Handlers | RESTful 接口，JWT 鉴权，业务逻辑编排 |
| **数据持久层** | PostgreSQL + pgvector | 用户、会话、消息、知识库、文档、向量块 |
| **外部服务层** | DeepSeek API / DashScope | LLM 推理、文本向量化 |

## 二、认证数据流 (Authentication)

```
┌──────────┐    POST /api/auth/login     ┌─────────────┐
│  Login   │ ───────────────────────────▶ │  login/     │
│  Form    │   { email, password }        │  route.ts   │
│ (zod     │                              │             │
│ 验证)    │ ◀─────────────────────────── │             │
└──────────┘   { user, JWT cookie }       └──────┬──────┘
                                                  │
                   POST /api/auth/register        │ bcrypt.compare()
┌──────────┐    { username, email, password }     │ jwt.sign(7d)
│ Register │ ───────────────────────────────────▶ │
│  Form    │                                      ▼
│ (zod     │                              ┌──────────────┐
│ 验证)    │                              │ PostgreSQL   │
└──────────┘                              │ users 表      │
                                          │ - id          │
      ┌──────────┐                        │ - username    │
      │  Logout  │  POST /api/auth/logout │ - email       │
      │  Button  │ ─────────────────────▶ │ - password_   │
      └──────────┘   clear cookie         │   hash(bcrypt)│
                          +                └──────────────┘
                     localStorage.clear()
                     redirect → /login
```

### 关键细节

1. **登录** ([app/api/auth/login/route.ts](app/api/auth/login/route.ts#L7-L63))：
   - 查询 `users` 表验证邮箱
   - `bcrypt.compare()` 验证密码哈希
   - `jwt.sign()` 签发 token，有效期 7 天
   - 设置 `auth_token` httpOnly cookie（`sameSite: "lax"`）

2. **注册** ([app/api/auth/register/route.ts](app/api/auth/register/route.ts))：
   - 检查邮箱唯一性
   - `bcrypt.hash(password, 10)` 加密
   - INSERT INTO `users`

3. **鉴权守卫**：每个 API 路由从 `req.cookies.get("auth_token")` 读取 JWT，`jwt.verify()` 解码出 `userId`，确保数据隔离

4. **登出**：清除 cookie + `localStorage.clear()` + 重定向到 `/login`

```
┌─────────────────────┐

│                      │     ┌─────────────────────┐
│ refetchInterval:     │     │ JWT 解码 → userId   │
│   title==="新对话"    │     │ 查询 chat_sessions   │
│   ? 2000ms           │     │ 查询 chat_messages   │

└──────────┬───────────┘     └─────────────────────┘
           │
           ▼
┌──────────────────────┐

│   messages           │

└──────────────────────┘
```

**关键设计**：当会话标题为 "新对话" 时，每 2 秒轮询一次 —— 这是为了等待后端异步的 AI 标题生成完成（`updateTitleAsync`）。标题一旦更新为非默认值，轮询自动停止。

### 4.3 查询时 RAG 检索

```
用户提问
    │
    ▼
┌──────────────────────┐
│ rewriteQuery()        │  ← buildRagContext.ts
│                       │
│ 调用 DeepSeek API     │
│ - model: deepseek-    │
│   chat               │
│ - 系统提示: 将口语化  │
│   问题改写为搜索关键词 │
│ - temperature: 0.3    │
│ - max_tokens: 60      │
│                       │
│ 例: "怎么让程序快点"   │
│  → "代码性能优化       │
│     程序加速方法       │
│     运行效率提升"     │
└──────────┬───────────┘
           │ rewrittenQuery
           ▼
┌──────────────────────┐
│ buildRagContext()     │
│                       │
│ ① embedSingle(query) │
│    → DashScope        │
│    1024维向量         │
│                       │
│ ② pgvector 检索:     │
│    SELECT dc.content, │
│           d.filename  │
│    FROM doc_chunks dc │
│    JOIN documents d   │
│    WHERE doc_id =     │
│      ANY($1)          │  ← 限定用户选中的文档
│    ORDER BY           │
│      embedding <=>    │  ← 余弦距离 (pgvector)
│      $2::vector       │
│    LIMIT 5            │  ← top-K
│                       │
│ ③ 拼接上下文:         │
│    "## 参考资料\n     │
│     ### 来自文档:     │
│       xxx.md\n        │
│     [chunk内容]\n     │
│     请基于以上参考... │
│     标注格式:         │
│     （根据《xxx.md》  │
│      文档）"          │
└──────────┬───────────┘
           │ ragContext 字符串
           ▼
   注入 system prompt
   → streamText()
```

### 5.2 React Query 缓存结构

```
QueryClient (staleTime: 5min, retry: 1)
```

### 5.3 状态流向图

```
用户操作
  │
  ├──▶ Zustand (UI 状态) ──▶ localStorage (持久化偏好)
  │     │                      - 模式切换、知识库选择等
  │     │
  │     └──▶ API 请求 body (deepThinking, knowledgeBaseId, documentIds)
  │
  ├──▶ React Query (服务端缓存)
  │     │
  │     ├── GET  → API Route → PostgreSQL → 缓存 → 组件渲染
  │     │
  │     └── MUTATION → 乐观更新缓存 → API Route → PostgreSQL
  │                    ├── 成功: invalidateQueries 同步
  │                    └── 失败: context.previous 回滚
```
