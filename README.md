# 1.智能文档助手系统

集成RAG（检索增强生成）技术，实现基于私有文档的精准问答的文档管理系统。(如果不清楚细节的可以参考一下飞书的知识回答)

参考资料：

https://docs.langchain.com/oss/javascript/langchain/overview

## 基础要求（必须完成）：

1. 文档上传
2. 文档管理
3. 文档列表展示、删除、重命名 
4.  RAG 问答能力，基于用户问题检索相关文档片段，结合上下文生成答案并展示引用来源 
5.  问答记录管理，能查看历史问答记录 
6.  基础页面功能 
   1.  登录 / 注册
   2.  404，403 页面 
   3.  路由懒加载

## **核心流程（尽量完成）**：

### 文档处理流程

1.  用户在主页上传文档 
2.  系统解析文档内容 
3.  对文本进行分块
4.  生成向量embedding 
5.  存入向量数据库 

### 问答流程

1.  用户输入问题 
2.  系统对问题生成向量 
3.  在向量库中检索相关片段（Top-K） 
4.  拼接上下文 Prompt 
5.  调用大模型生成答案 
6.  返回答案与引用来源

## 加分项（选择完成，可有可无）：

1. **检索缓存**：缓存机制优化性能，能对相同问题快速响应
2. **文档预览**：文档预览与编辑，类似office，wps（优先md）
3. **上传**：文件批量上传，进度显示，大文件分片...
4. **流式回答**：打字机效果输出答案，提升体验
5. **文档预处理**：表格、图片解析
6. **Query Rewrite**：提供AI润色改写功能，弥补用户口语提问和文档专业表述之间的语义鸿沟
7. **RAG进阶**：RAG效果的评估、知识库的动态更新...
8. 以及其他任何你能想到的亮点...
